# 汪曾祺式技术写作风格 (Wang Zengqi Writing Style)

本 Skill 旨在将生硬的技术文档转化为具有“汪曾祺式”美感的文字：短句为主、文字干净、逻辑理性但带有生活烟火气。

## 核心风格指南

1. **短句干净**：避免冗长的从句。能用一个逗号断开的，绝不连着写。
2. **去辞藻化**：不堆砌华丽的形容词。用最朴素的动词和名词描述技术动作。
3. **生活感（烟火气）**：将技术比作生活中的琐事，或者在叙述中加入一点点个人的观察和感受。
4. **理性打底**：文采是点缀，核心依然是清晰的技术逻辑。
5. **不拗口**：读起来要顺，像是在老树下喝茶聊天。

## 适用场景

- **技术原理**：化繁为简，讲得通透。
- **踩坑/实战记录**：加入一点点无奈或释然的情绪，让读者感同身受。
- **复盘/心得**：干练、有灵气，点到为止。

## 改写示例

### 原文 (生硬技术风)
> 为了解决 React 组件在频繁更新时产生的性能瓶颈问题，我们引入了 `useMemo` 钩子。通过对计算结果进行缓存，只有当依赖项发生变化时才重新执行逻辑，从而有效减少了不必要的渲染开销。

### 汪曾祺式改写
> React 组件更新快了，难免有些吃力。这时候，得用上 `useMemo`。其实就是存个档，省得每次都从头算起。只有等那几个关键的数变了，才重新动动手。这样一来，页面跑起来就轻快多了，不费那冤枉劲。

- "把这段踩坑经历写得有烟火气一点，别太生硬。"
