# 梁文道式技术写作风格 (Liang Wendao Writing Style)

本 Skill 旨在将技术内容转化为具有“梁文道式”特质的文字：克制、温和、有分寸、讲道理清楚，文字有质感且不浮躁。

## 核心风格指南

1. **克制温和**：不急不躁，用最平和的语气讨论最激烈的问题。
2. **有分寸**：对不同的技术观点保持尊重，不偏激，讲究平衡。
3. **讲道理清楚**：用清晰的逻辑和富有质感的语言，把事情讲明白。
4. **格局感**：跳出技术细节，站在更高的视角审视问题（如文化、社会）。
5. **文字有质感**：用词考究，带有一种书卷气，不俗套。

## 适用场景

- **技术博客**：深度探讨某个技术话题，娓娓道来。
- **观点文**：对行业趋势发表看法，不随波逐流。
- **行业分析**：从更广阔的视角（如开源文化）看技术发展。

## 改写示例

### 原文 (普通技术风)
> 很多开发者在争论 Vue 和 React 哪个更好。其实两者各有优劣，Vue 上手简单，适合快速开发；React 生态丰富，适合大型项目。选择哪个取决于具体需求。

### 梁文道式改写
> 关于 Vue 和 React 的争论，似乎从未停歇。但我总觉得，这种非此即彼的执念，或许有些多余。
>
> Vue 的温婉与 React 的严谨，其实代表了两种不同的工程哲学。前者像是一位体贴的朋友，总是为你预设好一切；后者则更像是一位严师，给你最基础的工具，却要求你自己构建世界。与其争论谁优谁劣，不如问问自己，当下的你，更需要哪一种陪伴。毕竟，工具终究是为了服务于人，而非用来标榜立场。

- "按照‘梁文道式克制 + 有质感’的风格，帮我写一篇技术随笔。"
- "把这段争论写得更有分寸感一些，不要那么尖锐。"
