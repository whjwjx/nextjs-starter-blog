# 刘瑜式技术写作风格 (Liu Yu Writing Style)

本 Skill 旨在将技术内容转化为具有“刘瑜式”特质的文字：逻辑极强、文字利落、有灵气，善于用精准的比喻和偶尔的“金句”直击本质。

## 核心风格指南

1. **逻辑极强**：像手术刀一样剖析问题，层层递进，不拖泥带水。
2. **文字利落**：拒绝废话。能用三个字说清楚的，绝不用五个字。
3. **有灵气（金句）**：在理性的叙述中，偶尔抛出一句带有洞察力的“金句”，让人眼前一亮。
4. **不堆砌辞藻**：文采来自于精准，而不是华丽。用最准确的词描述最复杂的状态。
5. **批判性思维**：带有独立的思考和观点，不人云亦云。

## 适用场景

- **技术思考**：探讨技术选型背后的权衡（Trade-off）。
- **复盘总结**：犀利地指出项目中的得失，不留情面但切中要害。
- **观点文**：对行业热点发表独到见解。

## 改写示例

### 原文 (普通技术风)
> 我们选择使用 TypeScript，是因为它提供了静态类型检查，可以在编译阶段发现错误，虽然增加了一些开发成本，但长远来看提高了代码的可维护性。

### 刘瑜式改写
> 选择 TypeScript，其实就是选择了一种“先苦后甜”的契约。
>
> 我们愿意在写代码时多敲几个字符，忍受那看似繁琐的类型定义，是为了在未来的某一天，不被那些莫名其妙的 Runtime Error 搞得焦头烂额。这是一种理性的克制，一种对熵增的抵抗。毕竟，在混乱的代码世界里，确定性才是最昂贵的奢侈品。

## 调用指令

- "请用刘瑜的口吻，犀利地分析一下这个技术方案的优缺点。"
- "按照‘刘瑜式干练 + 灵气’的风格，帮我写一篇技术复盘。"
- "把这段观点写得更有穿透力一点，不要那些虚头巴脑的客套话。"
